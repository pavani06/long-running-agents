# 📊 Team Progress Tracker
## Template Interativo para Rastreamento de Progresso da Equipe no Currículo Long-Running Agents

**Propósito:** Rastrear progresso individual e coletivo  
**Público:** Líderes de programa, mentores e participantes  
**Atualização Recomendada:** Semanal (sextas-feiras)  
**Status:** ✅ Template funcional — pronto para uso

---

## 📖 Prólogo: Por Que Rastrear Progresso?

O programa Long-Running Agents tem 12 semanas, 4 níveis e dezenas de exercícios. Sem um tracker visível, você perde de vista:

- Quem está atrasado e precisa de ajuda
- Quem está avançando rápido e pode mentorar
- Onde estão os gargalos da equipe (ex: todo mundo travado em Rubric Design?)
- Se o cronograma de 12 semanas é realista

**Este template resolve isso.** É uma ferramenta viva: você preenche, atualiza e usa para tomar decisões sobre alocação de mentoria, workshops extras e ajustes de cronograma.

Adapte o número de linhas e colunas conforme o tamanho da sua equipe. Os exemplos abaixo usam uma equipe fictícia de 5 pessoas.

## 🔧 Instruções de Uso

### Como Preencher Este Template

1. **Copie este arquivo** para o seu repositório de trabalho. Mantenha o original como referência.

2. **Substitua `[Nome]`** pelos nomes reais dos membros da sua equipe. Adicione ou remova linhas conforme necessário.

3. **Atualize os status semanalmente.** O ideal é sexta-feira, após cada membro reportar seu progresso. Use a legenda:
   - `✅` para módulos/exercícios concluídos
   - `🔄` para itens em andamento
   - `⏳` para itens não iniciados
   - `⚠️` para bloqueios (descreva o bloqueio nos Observações)
   - `🏆` para membros que dominam o nível e podem mentorar

4. **Preencha as métricas da equipe** com base nos dados da tabela de progresso. A seção de métricas oferece uma visão de alto nível para reportar a stakeholders.

5. **Atualize a timeline semanalmente** com marcos alcançados, membros destaque e observações relevantes.

6. **Use os checkpoints** para verificar se a equipe está no ritmo esperado. Se um checkpoint falhar, agende uma sessão de alinhamento.

### Como Adaptar para Diferentes Tamanhos de Equipe

- **Equipe pequena (2-3 pessoas):** Remova as linhas excedentes da tabela de progresso. As tabelas de detalhamento por nível também podem ser simplificadas — considere usar uma única tabela que cubra todos os níveis em vez de separar por nível.

- **Equipe média (4-8 pessoas):** O template está otimizado para este tamanho. Use como está.

- **Equipe grande (9+ pessoas):** Adicione linhas à tabela de progresso. Considere dividir a equipe em subgrupos com trackers separados para facilitar a leitura. Para as tabelas de detalhamento por nível, use o formato de matriz com nomes nas colunas — isso escala melhor que linhas por pessoa.

- **Múltiplos times:** Crie uma cópia deste arquivo para cada time. Use a seção de métricas agregadas para consolidar os dados de todos os times em um dashboard separado.

### Dicas para Líderes de Programa

- **Aja sobre os dados, não apenas colete.** Se o heatmap mostra 3 pessoas bloqueadas em Rubric Design, organize um workshop. Se a velocidade está abaixo do esperado, investigue a causa raiz.
- **Celebre marcos.** Quando alguém conclui um nível ou atinge o status 🏆, anuncie para a equipe. Reconhecimento mantém o engajamento.
- **Use os mentores.** Membros com 🏆 em um nível são seus melhores recursos para desbloquear quem está ⚠️.
- **Atualize a seção de métricas antes de reuniões de status.** Ela foi desenhada para ser copiada diretamente para apresentações ou reports.
- **Não espere perfeição nos dados.** O objetivo é ter visibilidade, não precisão cirúrgica. Se um membro está entre 60-70% em um módulo, marque 🔄 e siga em frente.

### Como Membros Devem Reportar Progresso

Cada membro deve responder estas 3 perguntas antes da atualização semanal:

1. **O que completei esta semana?** (módulos lidos, exercícios feitos, case studies estudados)
2. **O que está bloqueando meu progresso?** (conceito confuso, falta de tempo, dependência externa)
3. **Qual é meu plano para a próxima semana?** (próximos módulos, exercícios alvo, meta pessoal)

O líder consolida as respostas e atualiza o tracker.

## 🎓 O Que Você Aprendeu

Este template ensina, na prática, os princípios do programa:

- **Visibilidade substitui adivinhação.** Em vez de "achar" que a equipe está bem, você tem dados.
- **Checkpoints previnem o Context Amnesia.** Assim como um agente precisa de persistência de estado, uma equipe precisa de registros de progresso.
- **Métricas direcionam ação.** O heatmap de gargalos é o equivalente humano do trace reading — você identifica exatamente onde intervir.
- **Mentores são Evaluators.** Membros 🏆 revisam o trabalho de membros 🔄, aplicando o padrão Generator/Evaluator à dinâmica de equipe.
- **Templates escalam.** Uma equipe de 3 ou 30 pessoas usa a mesma estrutura — só muda o número de linhas.
