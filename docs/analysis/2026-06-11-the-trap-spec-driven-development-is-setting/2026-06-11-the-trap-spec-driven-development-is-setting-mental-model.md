## 1. Project Goals

- Build and manage long-running AI agent workflows through `.opencode/`, Node >= 20.18.0, ESLint, runtime state in `.runtime/` and `artifacts/`, and accepted decisions under `docs/decisions/` (`AGENTS.md:7`, `AGENTS.md:9`, `AGENTS.md:10`, `AGENTS.md:11`, `AGENTS.md:12`).
- Serve as a knowledge base and curriculum for AI systems that operate reliably for hours, days, or as long as a task requires without losing context, planning ability, or quality judgment (`README.md:3`).
- Teach the structural failure model for long-running agents: context loss, fragile planning, and blind self-evaluation, then teach harnesses as support structures that manage context, decompose work, and separate generation from evaluation (`README.md:7`, `README.md:9`, `README.md:10`, `README.md:11`, `README.md:13`; `curriculum/README.md:21`, `curriculum/README.md:27`).
- Target business builders with agentic-system skill, from beginners to production operators who want more reliable systems (`README.md:15`, `README.md:17`).
- Deliver a 12-week curriculum with 4 levels, 8 core concepts, 35+ diagrams, implementation guides, templates, and KODA-specific application (`curriculum/README.md:11`, `curriculum/README.md:13`, `curriculum/README.md:241`, `curriculum/README.md:264`; `docs/system-of-record.md:48`, `docs/system-of-record.md:50`).
- Operationalize repository work through issue claim, isolated worktree, execution brief, validation, draft PR, second-agent review, explicit merge approval, and cleanup (`.opencode/skills/issue-start/SKILL.md:12`, `.opencode/skills/issue-start/SKILL.md:14`, `.opencode/skills/issue-review/SKILL.md:12`, `.opencode/skills/issue-review/SKILL.md:14`, `.opencode/skills/issue-finish/SKILL.md:12`, `.opencode/skills/issue-finish/SKILL.md:14`).
- Maintain documentation governance through a strict authority chain: accepted ADRs, active canonical docs, validated evidence, analyses, archive docs, then READMEs and operational summaries (`docs/system-of-record.md:14`, `docs/system-of-record.md:16`, `docs/system-of-record.md:17`, `docs/system-of-record.md:18`, `docs/system-of-record.md:19`, `docs/system-of-record.md:20`, `docs/system-of-record.md:21`; `AGENTS.md:64`, `AGENTS.md:66`).

## Synthesis

The useful mental model is that `long-running-agents` is simultaneously a curriculum, an agent-work operating system, and a canonical pattern library. The curriculum teaches why long-running agents fail and how to harness them; `.opencode` operationalizes work through HoP agents and issue/review/finish skills; `docs/system-of-record.md` resolves source authority; `docs/canonical/` names reusable architectural patterns and their maturity gaps. Future phases for this source should compare extracted ideas only against these repository layers and should treat the external source as unexamined until Phase 1.
